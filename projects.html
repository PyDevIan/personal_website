Fridge Alert Notification System
Problem:

Industrial fridges must maintain strict temperature control for food safety. When a temperature breach is recorded by a fridge monitoring device, delays in response can cause product loss, compliance issues, or operational downtime. There was no automated alert system in place to track, assign, and escalate these events in real time.

Solution:

I developed an automated notification agent that monitors a dedicated inbox for alert emails sent by the fridge's industrial recorder. Upon detecting a new alert, it initiates a message loop in Microsoft Teams, notifying key roles (Factory Director, Head Mechanic, Automation Expert).

A FastAPI-powered web interface allows technicians to claim or acknowledge the alert, recording their action. If no one responds within a time window, the agent escalates the alert by re-notifying the team. The loop continues until a resolution email is detected (same device ID, alert marked inactive).

All alerts, acknowledgments, and responsible personnel are logged in a structured CSV file for auditability and trend analysis.

Invoice Extraction & Excel→ERP Pipeline
Problem:

The finance team receives heterogeneous invoices (Greek/English, mixed layouts) as PDFs and images. Manual keying into the company’s Excel template and then into the ERP is slow, error-prone, and hard to audit. We needed an automated, auditable pipeline that extracts structured fields and adds a human-in-the-loop validation step before bulk uploading into the ERP.

Solution:

I built a modular extraction service that combines PDF text parsing, OCR with layout awareness, and an LLM-based normalizer to produce a clean, strict schema (Supplier, Invoice Serial, Date, VAT/AFM, Total, TAX, Description, Cost Center). A Streamlit validator app lets authorized users review each row, stamp it as OK, and export only validated rows. An accuracy evaluator measures model quality against ground truth and guides iterative improvements. Finally, validated Excel batches are uploaded to the ERP in one shot.

How it Works (Pipeline):

  1. Ingestion — Watch an input folder for PDFs/PNGs. Prefer embedded PDF text; fallback to OCR with layout JSON. Cache all artifacts (raw text, OCR JSON, page thumbnails).
  2. Targeted ROI Corrections — For hard fields (VAT/AFM, totals, serial), locate labels/bounding boxes, crop regions on the saved page image, and re-OCR just those areas; run AFM checksum & amount normalization.
  3. LLM Normalization — Feed text + corrected hints into the model with a strict JSON contract; enforce types (ISO date, 2-decimal amounts, 9-digit VAT), then write a row to a master Excel file.
  4. Accuracy Evaluator — Optionally compare extracted rows against a ground-truth Excel using exact and fuzzy metrics per field plus row-level hybrid scoring. Export a mismatch report for training/threshold tuning.
  5. Human-in-the-Loop Validator (Streamlit) — Secure login; left-pane fields; right-pane PDF preview. Editors can adjust fields, select Cost Center, then stamp the row as OK. Only OK rows are eligible for export.
  6. Validated Export — Create a new Excel with OK & not yet exported rows. Mark those rows as EXPORTED in the master sheet (with user/date stamp) to prevent duplicates.
  7. ERP Batch Upload — The exported Excel follows the ERP’s import schema. A lightweight uploader script posts the file to the ERP endpoint (or drops it to a watched folder), logging responses for auditability.
